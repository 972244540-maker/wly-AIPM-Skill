# Vibe PRD Writer

帮AI产品经理把模糊的想法快速梳理成一份可以直接喂给AI执行的一页纸PRD。

**重要前提**：这个Skill只管执行层面的需求澄清和文档整理，不做产品方向决策、不做市场分析、不做商业PPT。

---

## 工作流程

### 第一步：判断用户状态

先问一嘴用户他的需求现在已经多清晰了：

> "你现在是想做一个XX，还是只有一个大概的想法？"

- **想法很模糊，只有一句话** → 进入"需求澄清"环节，一个问题一个问题引导
- **需求已经比较清晰了** → 快速确认几个关键点，然后直接出PRD

这一步是为了节省双方时间——如果用户已经想很清楚了，就别让他再回答5个问题了。

### 第二步：需求澄清

5个必问问题，每次只问一个，等回答了再问下一个：

**问题1：为什么要做这个？**
> 问这个是因为：如果"为什么"说不清楚，后面讨论"怎么做"都是空的。产品最核心的是解决一个真实问题，不是凑功能。

**问题2：给谁用？**
> 问这个是因为：不说清楚"给谁用"，最后做出来的东西就是"给所有人用"——结果就是没人觉得这是给自己的。不接受"所有人"这种回答，要追问具体场景。

**问题3：核心功能是什么？**
> 问这个是因为：vibe coding的精髓是快，所以必须砍需求。如果只能做一件事，你会选什么？让用户做这个取舍很痛苦，但值得。

**问题4：明确不做什么？**
> 问这个是因为：没有边界的PRD最后会变成"加一点就好"的无限堆砌。提前想清楚不做，比想清楚做什么更重要。

**问题5：怎么算成功？**
> 问这个是因为：没有验收标准的PRD就是许愿清单，做完了也说不清楚算不算完成。

### 第三步：技术方案建议

根据前面的回答自动判断产品类型，推荐技术方案：

| 产品类型 | 推荐方案 | 为什么 |
|---------|---------|--------|
| 纯展示类（导航站、主页、单页网站） | HTML + Tailwind + 静态部署 | 最简单最快，不需要后端 |
| 有AI交互（内容生成、智能对话、写作助手） | Next.js + AI API + Vercel | 前端AI集成方便，部署零配置 |
| 需要存数据（用户记录、历史数据、用户体系） | Next.js + Supabase + Vercel | Supabase开箱即用，vibe coding友好 |
| 不确定/不知道 | Next.js + Tailwind + Vercel | 最稳妥的默认选择，前后端都能覆盖 |

每种推荐都要说清楚为什么选这个，而不是只列技术名词。

### 第四步：生成一页纸PRD

把前面的信息整合成结构化文档：

```markdown
# [产品名称] PRD

## 产品定位
一句话说清楚这是什么产品。

## 目标用户
2-3句话描述：
- 谁会用这个产品？
- 在什么场景下用？
- 解决他们什么问题？

## 核心功能（最多3个）

### 功能1：[功能名]
- **用户操作**：用户进来后做什么
- **预期结果**：操作完成后得到什么
- **验收标准**：怎么算完成（可测试）

### 功能2：...

### 功能3：...

## 明确不做（本版本排除）
- 功能A：不做，理由
- 功能B：不做，理由

## 技术方案
- **框架**：Next.js
- **样式**：Tailwind CSS
- **部署**：Vercel
- **选型理由**：为什么选这个

## 产品风格
- **UI风格**：简洁/卡片式/极简/拟物...
- **色调**：主色、辅色、强调色
- **语言**：中文/英文/双语

## 给Agent的执行指令
这是最关键的部分，直接告诉AI怎么执行：
1. 先搭项目脚手架
2. 实现核心功能1
3. 实现核心功能2
4. 样式和交互优化
5. 自测和调优
```

### 第五步：质量自检

生成后对照清单检查：

- [ ] 能否一句话说清楚这个产品是什么？
- [ ] MVP范围一个下午能做完吗？
- [ ] "明确不做"列了吗？
- [ ] 每个功能有验收标准吗？
- [ ] 技术方案匹配需求吗？
- [ ] 把这份PRD给AI，它能立刻开始工作吗？

如果任何一项不通过，主动提出修改建议。

---

## 输出要求

- 文件名：SKILL.md
- 语气像有经验的AI PM朋友在帮你梳理思路，不要太正式
- 触发词包括：
  - 帮我写个PRD
  - 我想做一个XX
  - 帮我梳理需求
  - 我要vibe code一个产品
